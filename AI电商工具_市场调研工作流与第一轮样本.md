# AI 电商工具 市场调研工作流与第一轮样本

## 1. 这份调研要解决什么

这份文档不是为了泛泛介绍行业。

它要服务于一个具体目标：

```text
帮我判断 AI 电商运营工作台应该先模仿谁、学什么、做哪个最小闭环。
```

所以调研不按“品牌名气”排序，而按“它对我的 MVP 有没有参考价值”排序。

## 2. 我会怎么做市场调研

市场调研分 3 层。

### 第 1 层：资料抓取

目标：

```text
把官网、产品页、帮助中心、案例页里的公开信息抓下来。
```

重点资料：

- 官网产品介绍
- 产品中心
- 帮助中心
- 定价页
- 客户案例
- 官方博客
- 应用市场页面

不优先看：

- 泛泛媒体稿
- 没有具体功能的软文
- 只讲概念不讲流程的文章

### 第 2 层：流程拆解

每个品牌只回答 6 个问题：

```text
1. 它服务谁？
2. 它解决什么日常动作？
3. 用户输入什么？
4. AI 或自动化做了什么？
5. 输出是什么？
6. 输出怎么回到业务闭环？
```

如果这 6 个问题答不出来，说明这个品牌暂时不适合作为对标。

### 第 3 层：模仿判断

用本地已有的 `dbs-benchmark` 思路做判断。

只问 5 个问题：

```text
1. 它是否有明确商业价值？
2. 我能不能看懂它怎么提效？
3. 我能不能模仿其中一小段流程？
4. 它和我的 MVP 是否有关？
5. 它是否能帮助我做出可展示作品？
```

这里不是完整复制一个品牌，而是拆出最小可模仿环节。

## 3. 推荐使用的开源工具

### 首选：Crawl4AI

用途：

```text
把官网、帮助中心、产品页转换成适合 LLM 阅读的 Markdown。
```

为什么适合：

- 开源。
- 面向 LLM 数据清洗。
- 输入 URL，输出干净 Markdown。
- 适合做产品页、帮助中心、案例页采集。

项目：

```text
https://github.com/unclecode/crawl4ai
```

官方描述里明确说它能把网页转换成 LLM-ready Markdown，适合 RAG、Agent 和数据管线。

### 备选：Firecrawl

用途：

```text
网页搜索、抓取、结构化抽取。
```

为什么适合：

- 开源。
- 支持输出 clean Markdown。
- 支持按 JSON schema 抽取结构化字段。
- 更适合以后做自动化调研工具。

项目：

```text
https://github.com/firecrawl/firecrawl
```

### 深度研究：Open Deep Research

用途：

```text
自动搜索、迭代研究、生成长报告。
```

为什么适合：

- 适合做“一个主题的多轮研究”。
- 可以用于后续深挖一个品牌或一个细分方向。

项目：

```text
https://github.com/langchain-ai/open_deep_research
```

### 工作流编排：Dify

用途：

```text
把“抓页面 -> 提取字段 -> 输出表格 -> 生成结论”做成可重复工作流。
```

为什么适合：

- 可视化 workflow。
- 适合非工程化阶段快速搭调研 Agent。
- 后续可以把市场调研流程固化成工具。

项目：

```text
https://github.com/langgenius/dify
```

## 4. 第一轮品牌样本

第一轮不追求全覆盖，先看 6 个和当前 MVP 最相关的对象：

```text
即创
有赞
微盟 WAI
妙手
店小秘
影刀
```

它们覆盖 4 种类型：

```text
内容生成型
商家经营型
跨境 ERP 型
RPA 自动化型
```

## 6. 对我的 MVP 的直接启发

### 即创给我的启发

可模仿点：

```text
商品信息 -> 平台化内容生成
```

对应我的 MVP：

```text
商品资料 -> 小红书文案 / 抖音脚本 / 直播话术
```

我不需要一开始做视频生成，只需要先做脚本和文案。

### 有赞给我的启发

可模仿点：

```text
AI 不只是生成内容，还要回到经营动作。
```

对应我的 MVP：

```text
经营数据 -> AI 复盘 -> 下一步行动清单
```

这说明“复盘建议”不是可有可无，而是把工具从文案生成拉到经营助手的关键。

### 微盟 WAI 给我的启发

可模仿点：

```text
围绕商家日常场景做模板化 AI。
```

对应我的 MVP：

```text
商品描述
种草笔记
直播口播稿
客服话术
```

第一版应该重视模板和输出结构，不要只给一大段自由文本。

### 妙手给我的启发

可模仿点：

```text
AI 输出要进入商品运营流程，而不是停在聊天框。
```

对应我的 MVP：

```text
生成标题、描述、图片建议、短视频脚本后，能导出成素材包。
```

妙手的价值不只是 AI 生成，而是和采集、刊登、订单、物流等 ERP 流程连在一起。我的第一版不做 ERP，但要保留“导出素材包”的意识。

### 店小秘给我的启发

可模仿点：

```text
跨平台、多店铺、批量运营才是电商工具的长期价值。
```

对应我的 MVP：

```text
第一版只做单商品，但输出结构要便于以后扩展成批量商品。
```

暂时不做批量，但表单和结果结构不要写死到一个平台。

### 影刀给我的启发

可模仿点：

```text
真正提效不只是生成，而是自动执行重复操作。
```

对应我的 MVP：

```text
第一版先生成素材，下一版可以考虑“按步骤指导用户执行”。
```

我现在不做 RPA，但要意识到：商家最终想省掉的是“重复执行”。

## 7. 当前最值得模仿的闭环

第一版不要模仿全套品牌。

最值得模仿的是这条小闭环：

```text
商品资料
-> 卖点分析
-> 平台化内容生成
-> 客服话术生成
-> 经营数据复盘
-> 下一轮内容优化
```

对应参考对象：

```text
即创：内容生成
有赞：经营复盘
微盟：场景模板
妙手：素材进入商品运营
影刀：后续自动化方向
```

## 8. 第一版市场判断

这个方向值得先做 MVP，不是因为它一定能做成大公司，而是因为它满足三个条件：

```text
1. 用户痛点明确：商家每天都要写内容、回客户、看数据。
2. AI 可替代一部分工作：卖点提炼、内容初稿、客服话术、复盘建议。
3. 输出结果能被直接使用：复制、发布、导出、交给客服或运营。
```

但第一版不要假设它一定能赚钱。

第一版只验证：

```text
一个商品 -> 一份运营素材包 -> 用户愿不愿意用
```

## 9. 建议的调研工作流

后续每研究一个新品牌，都按这个流程。

```text
输入：品牌名 + 官网链接
   ↓
抓取：官网 / 产品页 / 帮助中心 / 案例页
   ↓
抽取：用户、场景、输入、AI动作、输出、闭环、收费
   ↓
判断：是否和我的 MVP 有关
   ↓
输出：一页品牌拆解表
```

## 10. 调研模板

```text
# 品牌调研：{品牌名}

## 1. 基本定位
它是什么工具？
服务哪类用户？

## 2. 核心场景
它解决用户哪个日常动作？

## 3. 输入
用户需要提供什么？

## 4. AI / 自动化处理
系统具体做了什么？

## 5. 输出
用户最后拿到什么？

## 6. 业务闭环
输出怎么回到真实业务里？

## 7. 收费或商业化
它怎么收费？
如果官网没写，写“待确认”。

## 8. 可模仿点
我能模仿哪一小段？

## 9. 不适合模仿的点
哪些太重、太复杂、太依赖平台资源？
```
