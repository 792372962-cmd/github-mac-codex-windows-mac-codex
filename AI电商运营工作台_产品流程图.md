# AI 电商运营工作台 产品流程图

## 1. 主流程

用户先输入商品资料，系统先提炼卖点，再按平台生成内容，最后输出可复制的运营素材。

```text
┌──────────────┐
│  输入商品资料  │
│ 商品名/价格/   │
│ 人群/卖点/场景 │
└──────┬───────┘
       │
       v
┌──────────────┐
│  AI 卖点分析   │
│ 提炼核心卖点   │
│ 用户痛点/理由  │
└──────┬───────┘
       │
       v
┌──────────────┐
│  选择平台      │
│ 小红书/抖音/   │
│ 朋友圈/详情页  │
└──────┬───────┘
       │
       v
┌──────────────┐
│  AI 内容生成   │
│ 标题/正文/脚本 │
│ 直播话术/详情页 │
└──────┬───────┘
       │
       v
┌──────────────┐
│  输出素材包    │
│ 复制 / 导出    │
└──────────────┘
```

主流程要解决的事情：

- 把商品资料变成可发布内容。
- 把内容生成从“人工写”变成“AI 先出初稿”。
- 让用户能直接复制到小红书、抖音、朋友圈、商品详情页。

## 2. 客服流程

用户先输入商品信息，系统生成常见问题和标准回复，帮助商家减少重复解释。

```text
┌──────────────┐
│  输入商品资料  │
│ 功能/价格/售后 │
└──────┬───────┘
       │
       v
┌──────────────┐
│  生成常见问题  │
│ 用户会问什么   │
│ 用户会担心什么 │
└──────┬───────┘
       │
       v
┌──────────────┐
│  生成客服回复  │
│ 标准回答/异议  │
│ 催单/售后解释  │
└──────┬───────┘
       │
       v
┌──────────────┐
│  输出话术包    │
│ 复制到客服使用 │
└──────────────┘
```

客服流程要解决的事情：

- 提前回答用户最常见的问题。
- 统一回复口径，减少客服临场发挥。
- 把“问一次答一次”变成“先生成一套标准话术”。

## 3. 复盘流程

用户输入经营数据，系统根据结果判断问题在哪一层，并给出下一步优化建议。

```text
┌──────────────┐
│  输入经营数据  │
│ 曝光/点击/     │
│ 加购/成交/退款 │
└──────┬───────┘
       │
       v
┌──────────────┐
│  AI 复盘分析   │
│ 判断转化瓶颈   │
│ 找出问题层级   │
└──────┬───────┘
       │
       v
┌──────────────┐
│  输出优化建议  │
│ 内容/卖点/价格 │
│ 人群/话术调整  │
└──────┬───────┘
       │
       v
┌──────────────┐
│  下一步行动    │
│ 修改内容后再测 │
└──────────────┘
```

复盘流程要解决的事情：

- 让用户知道问题不是“数据差”，而是具体卡在哪一步。
- 把经营数据变成可执行建议。
- 帮用户形成“发内容 -> 看数据 -> 改内容”的闭环。

## 4. 三条流程的关系

```text
商品资料输入
   │
   ├─> 主流程：生成内容素材
   │
   ├─> 客服流程：生成问答话术
   │
   └─> 复盘流程：根据数据给优化建议
```

三条流程共享同一份商品资料，但输出不同：

- 主流程输出内容。
- 客服流程输出话术。
- 复盘流程输出建议。
