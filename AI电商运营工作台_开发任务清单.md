# AI 电商运营工作台 开发任务清单

## 第 1 天：产品流程

### 任务

- 梳理主流程、客服流程、复盘流程。
- 确认 MVP 只做商品资料输入、卖点分析、内容生成、客服问答、经营复盘。
- 画出单页工作台结构。
- 明确哪些功能不做。

### 产物

- `AI电商运营工作台_产品流程图.md`
- 产品边界说明
- 页面结构草图

### 验收标准

- 三条流程说得清楚。
- 每条流程都能回答“输入是什么、输出是什么、用户拿来干什么”。
- 不包含登录、支付、订单、库存、数据库。

## 第 2 天：前端页面

### 任务

- 搭出单页工作台框架。
- 实现左侧导航。
- 实现中间输入区。
- 实现右侧输出区。
- 实现底部导出复制区。

### 产物

- 页面骨架
- 模块切换能力
- 基础表单
- 输出展示区

### 验收标准

- 页面能在一个屏幕内完成主要操作。
- 左侧切换时，中间和右侧能跟着切换模块。
- 页面不依赖后端也能跑起来。

## 第 3 天：接 AI

### 任务

- 接入卖点分析能力。
- 接入内容生成功能。
- 接入客服问答生成功能。
- 接入经营复盘建议生成。

### 产物

- 四类 AI 输出结果
- 基础提示词
- 生成结果展示逻辑

### 验收标准

- 输入商品资料后能生成卖点分析。
- 选择平台后能生成对应内容。
- 输入经营数据后能生成复盘建议。
- 输出结果结构固定，方便复制。

## 第 4 天：体验优化

### 任务

- 加载状态。
- 重新生成按钮。
- 复制按钮。
- 导出 Markdown。
- 示例数据填充。
- 输入输出分区优化。

### 产物

- 更好用的工作台交互
- 可直接复用的示例商品
- 输出整理格式

### 验收标准

- 用户能用示例商品直接跑一遍。
- 生成结果可以一键复制。
- 导出内容是整洁的 Markdown。
- 页面不会因为内容过长而乱掉。

## 第 5 天：作品集包装

### 任务

- 写项目说明。
- 补产品定位。
- 补页面截图。
- 补使用说明。
- 补简历表达。

### 产物

- 项目 README
- 产品介绍文档
- 页面截图
- 作品集描述

### 验收标准

- 外人能看懂你做了什么。
- 能清楚说出这个项目解决什么问题。
- 能用于 GitHub 私有仓库或作品集展示。
- 能作为后续继续开发的基础版本。

## 每天的执行原则

- 每天只推进一个明确目标。
- 不扩功能。
- 不碰登录、支付、订单、库存、数据库。
- 不做“顺手加一个别的功能”。
- 先把最小闭环跑通，再谈增强。
